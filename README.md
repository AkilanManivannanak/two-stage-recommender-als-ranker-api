## ML Pipeline вҖ” 5 Stages

### Stage 1 вҖ” PySpark Feature Engineering

```
MovieLens ratings (800k rows Г— 8 cols)
        в”Ӯ
        в–ј
PySpark ETL  (spark_features.py)  вҖ”  local[*] mode
Columnar groupBy aggregations faster than Python dict loops at this scale

5 feature sets computed:
  1. user_genre_ratings   вҖ” {uid: {genre: [ratings]}}
  2. user_activity        вҖ” {uid: {n_ratings, avg_rating, n_genres}}
  3. impression_counts    вҖ” {uid: {item_id: n_impressions}}
  4. item_popularity      вҖ” {item_id: interaction_count}
  5. item_cooccurrence    вҖ” {item_id: [top-10 co-watched items]}
                            via PySpark self-join on user_id

Fallback: if PySpark not installed вҶ’ pandas/dict implementation
          pipeline never hard-fails
```

### Stage 2 вҖ” ALS Collaborative Filtering + RAG

**ALS (Scala Spark MLlib)**
```
rank       = 64 latent factors
iterations = 20
alpha      = 40  (implicit feedback weighting)
вҶ’ item_factors stored in MinIO В· loaded into Redis
```

**Why Scala over PySpark?** Native Spark MLlib eliminates JVMвҶ”Python serialisation bridge, achieving 2вҖ“4Г— speedup per iteration.

**RAG Semantic Retrieval (Qdrant)**
```
Voice query вҶ’ OpenAI text-embedding-3-small (1,536-dim)
вҶ’ Qdrant HNSW cosine similarity
вҶ’ year вүҘ 1970 filter for "similar to X" queries
вҶ’ 3-strategy retry: exact title вҶ’ without year вҶ’ first 3 words
```

### Stage 5 вҖ” Slate Optimizer

```
Hard constraints (slate_optimizer_v2.py):

  вң“  вүҘ 5 distinct genres on assembled page       (MIN_GENRES_ON_PAGE = 5)
  вң“  вүӨ 3 titles from same genre in top-20 slate  (MAX_SAME_GENRE_IN_SLATE = 3)
```

## Offline RL / Off-Policy RL Evaluation

### REINFORCE Policy Gradient

```python
# session_intent.py вҖ” GRU-style session encoder
HIDDEN_DIM = 16   # GRU hidden state dimension  (actual value from code)
INPUT_DIM  = 8    # per-event feature dimension  (actual value from code)
# Single GRU cell В· numpy (no PyTorch dependency)
# h_t = GRU(x_t, h_{t-1})
# Trained on session sequences вҶ’ acc=0.927 reported at startup

# rl_policy.py вҖ” REINFORCE agent
# Monte Carlo returns: G_t = ОЈ Оі^k * r_{t+k}
# Policy gradient: вҲҮJ(Оё) = ОЈ G_t * вҲҮlog ПҖ(a_t|s_t)
# Policy weights stored in Redis В· updated per completed episode

Reward signal (reward_model.py, weights _W):
  play_start    вҶ’ +0.42 weight

  genre_match   вҶ’ +0.38 weight
```

### LinUCB Off-Policy Bandit

```python
# bandit_v2.py
# 8 genre arms: Action, Comedy, Drama, Horror,
#               Sci-Fi, Romance, Thriller, Documentary

class LinUCBArm:
    context_dim: int = 8    # feature dimension
    alpha:       float = 1.0  # exploration-exploitation tradeoff

    def ucb_score(self, context):
        # UCB = Ој(arm) + Оұ Г— вҲҡ(x^T A^{-1} x)
        exploit = theta @ context
        explore = alpha * sqrt(context @ A_inv @ context)
        return exploit + explore

# This IS off-policy RL:
# - learns from logged interactions under previous policies
# - updates per arm without live re-exploration
# - Thompson Sampling available as alternative strategy
```

## CLIP вҖ” Vision Transformer (ViT-B/32)

CLIP encodes visual features using a **Vision Transformer (ViT-B/32)** backbone with patch embeddings and multi-head self-attention. The image is split into 32Г—32 patches, each embedded into a 512-dim token; these tokens pass through transformer layers with self-attention before projection into the shared text-image embedding space.

```
Movie poster вҶ’ 32Г—32 patches вҶ’ 512-dim patch tokens
вҶ’ Transformer layers (multi-head self-attention)
вҶ’ [CLS] token вҶ’ 512-dim visual embedding
вҶ’ projected into CLIP shared space (aligned with text queries)

Graceful fallback (context_and_additions.py):
  when openai-clip not installed вҶ’ colour histogram fallback
  zero impact on ALS+LightGBM+RL core recommendations
```

## SQL Schema & Analytics

**`sql/schema.sql`** вҖ” 4 tables:

```sql
CREATE TABLE users (
    user_id         BIGINT PRIMARY KEY,
    profile_name    VARCHAR(64) DEFAULT 'Cinephile',
    activity_decile SMALLINT CHECK (activity_decile BETWEEN 1 AND 10),
    top_genres      TEXT[] DEFAULT '{}'
);

CREATE TABLE ratings (
    user_id  BIGINT REFERENCES users(user_id),
    item_id  BIGINT NOT NULL,
    rating   NUMERIC(3,1) CHECK (rating BETWEEN 0.5 AND 5.0),
    watch_pct NUMERIC(5,2)
);

CREATE TABLE recommendations (
    user_id         BIGINT REFERENCES users(user_id),
    item_id         BIGINT NOT NULL,
    rank            SMALLINT NOT NULL,
    als_score       NUMERIC(8,6),
    rl_score        NUMERIC(8,6),
    policy_version  VARCHAR(32) DEFAULT 'v6.0.0'
);

CREATE TABLE events (
    user_id     BIGINT REFERENCES users(user_id),
    item_id     BIGINT NOT NULL,
    event_type  VARCHAR(32) NOT NULL,  -- play_start | watch_90pct | skip | add_to_list
    reward      NUMERIC(4,2),
    session_id  UUID
);
```

**`sql/queries.sql`** вҖ” analytical queries (SELECT + JOIN + GROUP BY):

```sql
-- NDCG@10 per policy version
SELECT r.policy_version,
       COUNT(DISTINCT r.user_id) AS unique_users,
       AVG(CASE WHEN e.event_type = 'play_start' THEN 1.0
                ELSE 0.0 END / LOG(2, r.rank + 1)) AS ndcg_at_10
FROM recommendations r
LEFT JOIN events e ON e.user_id = r.user_id AND e.item_id = r.item_id
WHERE r.rank <= 10
GROUP BY r.policy_version
ORDER BY ndcg_at_10 DESC;

-- CTR by activity decile (SELECT + JOIN + GROUP BY + HAVING)
SELECT u.activity_decile,
       COUNT(DISTINCT u.user_id) AS users,
       COUNT(CASE WHEN e.event_type = 'play_start' THEN 1 END) * 100.0
       / NULLIF(COUNT(DISTINCT r.rec_id), 0) AS ctr_pct
FROM users u
JOIN recommendations r ON r.user_id = u.user_id
LEFT JOIN events e ON e.user_id = u.user_id AND e.item_id = r.item_id
GROUP BY u.activity_decile
HAVING COUNT(DISTINCT u.user_id) >= 10
ORDER BY u.activity_decile;
```

|---|---|---|---|
| 1 | "Dune is Romance" | `/explain` API used `user.top_genre` instead of `item.primary_genre` | `buildExplanation()` reads item fields directly вҖ” no API call |
