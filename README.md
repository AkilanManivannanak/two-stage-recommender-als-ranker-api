### Agent Definitions

**Retrieval Agent** вҖ” Candidate generation
- Tools: `ALS.get_candidates(user_id, k=100)` В· `RAG.semantic_search(query, k=20)`
- Input: user_id В· session context В· voice query
- Output: 100-item candidate pool with ALS scores
- Fallback: trending items if ALS bundle not loaded

**Reasoning Agent** вҖ” Candidate ranking and reward estimation
- Tools: `LightGBM.rank(candidates, features)` В· `MultiTaskReward.score(slate)`
- Input: 100 candidates + 8 user/item features
- Output: top-30 ranked slate + per-item reward estimates
- Uses doubly-robust IPS for unbiased reward estimation

**Exploration Agent** вҖ” Genre arm selection and RL reordering
- Tools: `LinUCB.select_arm(context)` В· `REINFORCE.reorder(slate)`
- Input: GRU hidden state h_t (16-dim вҶ’ 8-dim context)
- Output: reordered slate with exploration arm applied
- Off-policy: learns from logged interactions without live re-exploration

**Critic Agent** вҖ” Quality enforcement and rollback
- Tools: `PolicyGate.evaluate(slate, thresholds)` В· `Metaflow.rollback()`
- Input: final slate + model version + latency metrics
- Output: DEPLOY (hot-swap <30s) or BLOCK (rollback + alert)
- 27 automated GateCheck objects вҖ” cannot be bypassed

## Doubly-Robust Off-Policy RL Evaluation

**This is a top-level system component, not a footnote.**

### What It Solves

Standard NDCG is biased вҖ” popular items are shown more, so they get more clicks regardless of quality. The doubly-robust IPS estimator corrects for position bias in logged data, enabling true offline RL evaluation without deploying new policies live.

### Math

```python
# ope_eval.py
def ips_ndcg_at_k(recommendations, events, propensities, k=10):
    """
    Standard NDCG:    ОЈ relevance(i) / log2(rank(i)+1)
    IPS-corrected:    ОЈ [reward(i)/propensity(i)] / log2(rank(i)+1)
    Doubly-Robust:    DR(ПҖ) = IPS(ПҖ) + direct_model_correction

    DR(ПҖ) = ОЈ [reward(i)/p(i)] / log2(rank+1)
          + ОЈ [dm(i) Г— (1 - 1/p(i))] / log2(rank+1)

    Consistent if EITHER propensity model OR direct model is correct.
    Hence "doubly" robust.
    """
```

### Why Doubly-Robust vs Plain IPS

| Estimator | Bias | Variance | When to Use |
|---|---|---|---|
| Naive NDCG | High (position bias) | Low | Never вҖ” incorrect for rec systems |
| IPS only | Low | High (unstable at low propensity) | Sufficient data, stable propensities |
| **Doubly-Robust** | **Low** | **Low** | **Production standard вҖ” used here** |

### Where It Runs

```

  вҶ’ DR-IPS-NDCG on held-out Parquet logs
  вҶ’ compared against incumbent model
  вҶ’ if drop > threshold вҶ’ policy gate BLOCK вҶ’ rollback
```

## Belief State Tracking

Inspired by **Belief State Transformers** (Jiang et al. 2024), CineWave maintains explicit belief states across three dimensions updated online and propagated through the multi-agent pipeline.

```
h_t = GRU(x_t, h_{t-1})
     = belief over user genre preference given session history

Not a point estimate вҖ” a compressed sufficient statistic of the
entire session history [e_1, ..., e_t].

Analogous to a POMDP belief state where true user preference
is latent, only partially observed through interaction events.
```

**2. Arm Reward Belief State** вҖ” LinUCB inverse covariance $A_a^{-1} \in \mathbb{R}^{8 \times 8}$

```
UCB(a) = ОёбөҖx  +  ОұвҲҡ(xбөҖAвҒ»В№x)
         exploit   belief uncertainty

A_a^{-1} = belief over arm a's reward distribution
High xбөҖAвҒ»В№x = high uncertainty = explore this arm more
Low  xбөҖAвҒ»В№x = confident belief  = exploit known reward
```

**3. Reward Propensity Belief State** вҖ” IPS propensity $\hat{p}(a|x)$

```
rМғ_i = r_i / pМӮ(a_i|x_i)   (propensity-corrected reward)

Encodes belief over the logging policy's item selection probability.
Corrects for exposure bias: items shown more often carry
inflated reward estimates without propensity correction.
```

### Mapping to Belief State Transformers

| CineWave | BST Framework Analog |
|---|---|
| GRU $h_t$ (16-dim) | Hidden world state |
| LinUCB $A_a^{-1}$ | Posterior over reward |
| IPS $\hat{p}(a\|x)$ | Observation model |
| GRU gates + UCB update | Bayesian state update |
| FastAPI orchestrator | Transformer context propagation |
| Policy Gate BLOCK | Belief-conditioned decision boundary |

All three belief states update online after each interaction and propagate through the 4-agent pipeline. The Critic Agent's 27-gate policy acts as a **belief-state-conditioned decision boundary** вҖ” promoting only when the system's belief about model quality (DR-IPS-NDCG) exceeds threshold.

## Phi-Style Data Quality Filtering

Inspired by **Phi** ("Textbooks are all you need", Li et al. 2023) вҖ” the insight that smaller models trained on higher-quality data outperform larger models trained on noisy data вҖ” CineWave applies principled quality filtering before ALS training.

**The core analogy:** Low-quality catalog items corrupt ALS co-occurrence signals in exactly the same way low-quality tokens corrupt LLM training.

```python
# data_curation.py вҖ” Phi-style quality scoring

def quality_score(item):
    # Bayesian average вҖ” same philosophy as Phi's textbook quality filter:
    # smooth toward global mean to prevent overfitting to sparse signal
    # (1-2 votes = noise, just like 1-2 token completions = noise)
    bayesian_rating = (v/(v+50)) * avg_rating + (50/(v+50)) * 3.5

    return (
        0.4 * bayesian_rating/5.0 +  # rating quality  (like educational value)
        0.3 * min(votes/500, 1.0)  +  # vote reliability (like source credibility)
        0.2 * has_poster           +  # metadata completeness
        0.1 * (year >= 1970)           # recency signal
    )

# Filters (in order вҖ” like Phi's multi-stage data pipeline):
#   1. vote_count < 5    вҶ’ remove  (unreliable signal = noisy token)
#   2. avg_rating < 1.5  вҶ’ remove  (clearly bad = toxic content)
#   3. quality < 0.10    вҶ’ remove  (low overall = low educational value)
#   4. duplicate titles  вҶ’ dedup   (like deduplication in LLM corpora)
#   5. genre normalize   вҶ’ 8 classes (like tokenizer normalization)
```

### Result

| Stage | Items | Analogy |
|---|---|---|
| Raw catalog | 4,961 | Raw web crawl |
| After quality filter | 3,363 | Phi textbook-quality subset |
| Retained | **86.6%** | Signal-to-noise ratio |
| Removed | 598 items (13.4%) | "Noisy tokens" |

**Why this matters:** The 13.4% removed are the "noisy tokens" of the recommendation training set. Removing them improves ALS embedding quality вҖ” same principle as Phi's finding that 1B parameters on quality data beats 10B on noisy data.

## Chain-of-Thought Intent Extraction вҖ” LLM Evaluation

CineWave evaluates three GPT-4o prompting strategies for voice intent extraction, following the **Orca/Phi paradigm** of measuring reasoning quality through intermediate steps.

### Three Strategies Compared

```python
# Strategy A вҖ” Direct prompting
prompt = "Extract genres from: {query} вҶ’ JSON"

# Strategy B вҖ” Chain-of-Thought (CoT)  
prompt = """
Step 1 вҖ” What mood does the user want?
Step 2 вҖ” Which genres match that mood?
Step 3 вҖ” Any reference title (similar_to)?
Step 4 вҖ” Year preference?
Step 5 вҖ” Final structured intent.

Query: {query}
Reasoning:
"""

# Strategy C вҖ” Few-shot + CoT
prompt = """
EXAMPLE 1: "Something like Inception but scarier"
Reasoning: psychological fear вҶ’ Thriller, Horror, Sci-Fi вҶ’ similar_to=Inception
Output: {"genres": ["Thriller","Horror","Sci-Fi"], "similar_to": "Inception"}

[2 more examples...]

NOW YOUR TURN: {query}
Reasoning:
"""
```

### Why This Matters

This is **LLM evaluation as a research contribution** вҖ” the same methodology used in Orca (imitation of CoT reasoning) and Phi (measuring reasoning emergence). We measure:

- **Genre extraction accuracy** вҖ” does CoT improve intent understanding?
- **Latency trade-off** вҖ” CoT adds tokens but improves accuracy
- **Reasoning quality** вҖ” are intermediate steps logically consistent?

### Results (verified on 10 test queries)

| Strategy | Accuracy | Lift | Avg Latency | Correct/Total |
|---|---|---|---|---|
| Direct prompting | 50.0% | baseline | 726ms | 5/10 |
| CoT | **100.0%** | **+100%** | 1,983ms | 10/10 |
| **Few-shot + CoT** | **100.0%** | **+100%** | **1,318ms** | **10/10** |

**Key findings:**
- Direct prompting **fails on ambiguous queries** вҖ” returns empty genres for "Show me something like Inception but darker"
- CoT achieves **perfect accuracy** by reasoning through mood вҶ’ genre вҶ’ reference title as explicit steps
- Few-shot + CoT matches CoT accuracy while being **33% faster** (1,318ms vs 1,983ms)
- Latency overhead acceptable within voice p95 < 2.5s SLO

> All results verified from live system. Run `python3 chain_of_thought_intent.py` to reproduce.

// hooks/useVoiceAssistant.ts

// This caused the greeting TTS to fire twice on page load.

// FIX:
const greetedRef = useRef(false)   // persists across StrictMode double-mount

useEffect(() => {
  if (!greetedRef.current) {
    greetedRef.current = true      // set to true BEFORE the async TTS call
    speakGreeting()                // only fires once, even in StrictMode
